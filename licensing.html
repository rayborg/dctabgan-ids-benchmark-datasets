Conservative release posture

Licensing and provenance caveats

This release is intentionally conservative. It does not claim to relicense upstream corpora, and it does not assume that a public mirror automatically authorizes downstream public redistribution.

Where status is recorded per task

Every task row in metadata/datasets.csv and metadata/datasets.json includes downloadable, public_bundle_status, public_csv, omission_reason, omission_caveat, and redistribution_caveat. The downloadable task catalog repeats the same status next to each direct CSV link.

Downloadable means the public release includes a CSV under data/. Omitted/not redistributed means no public CSV is shipped here; the task is retained only as metadata plus a local/private recreation workflow.

Why mirrors matter

A public mirror can be useful for reproducibility, but it does not automatically prove that every downstream redistribution is authorized. This matters most when the benchmark used a mirrored artifact instead of an official open release or when the official route is gated and the public mirror carries separate terms.

For practical use, start with Download or Recreate Omitted.
